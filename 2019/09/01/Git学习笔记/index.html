Git学习笔记

发表于 2019-09-01 | 分类于 Git

该篇Blog主要是记录学习Git过程中所使用到的一些操作命令，以方便以后查找。强烈建议初学者参考廖雪峰的Git教程。

git与svn区别

最大不同是，使用git，在没有网络的情况下，你可以提交给本地（即本地服务器），在本地做版本管理的所有处理。比如提交版本1，提交版本2，提交3，然后恢复版本1，开一个分支，切换分支，修改代码，合并分支。这和集中式的svn非常不同，svn你离线时可以提交吗？绝对不能。git则可以在无网络情况下本地提交，他把提交版本（即提交到本地服务器）与推送服务器（即中央服务器）这两个概念分开了。而svn之流是融为一体的，提交即推送给服务器（即中央服务器）。因此，git是去中心的，他的服务器并非必须的，只是为了方便大家交流使用，你完全可以把git中央服务器想象成一个参与者，他得电脑始终在线，他不写代码，只接受大家的推送与合并，然后供其他用户从他这里拉取信息。

git三个分区

工作区：实际目录，不包括.git
暂存区: 版本库里存了很多东西，其中最重要的就是称为stage（或者叫index）的暂存区
版本库：隐藏目录.git，最终push也是推送这个版本

git命令

git init:在当前目录初始化git仓库，生成 .git 目录
git add 文件名：将文件添加到本地服务器（暂存区）
git commit -m “xxx” ：将多次修改增加删除的文件提交到本地服务器
git status：查看文件提交状态

【两种比较】

  • git diff 文件名 :比较的是当前分支的工作区和当前分支暂存区的差别(若git add后没有修改，就没有差别)
  • git diff HEAD -- 文件名:比较的是当前分支工作区和当前分支版本库的差别
  • git diff 别的分支名 -- 文件名:比较的是当前分支工作区和别的分支版本库的差别

git log [--pretty=oneline]：commit提交历史记录
git reflog：历史命令，查看commit号
git reset --hard [0xXXXXX]：回退到哪个commit版本
git reset --hard HEAD[^...]：回退上第几个版本

【四种撤销】
命令git checkout -- readme.txt意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：

  • readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态; —> (工作区和版本库一模一样)
  • readme.txt已经添加到暂存区后，”又作了修改”，现在，撤销修改就回到添加到暂存区后的状态。 —> （工作区git add（但未git commit）后又修改，回到修改前的，工作区和暂存区一模一样）

总之，就是让这个文件回到最近一次git commit或git add时的状态。还有下面两种撤销：

  • git reset HEAD <file>：可以把暂存区的修改撤销掉（unstage）（同时git reset可以回退版本），但是没有撤销工作区，再使用git checkout -- <file>：把工作区也撤销 —> （撤销暂存区和工作区到以前那样）
  • git add 和git commit后，使用git reset --hard 回退到具体版本。

【删除文件】

  1. 先删除工作区文件:删除工作区文件。
  2. git rm 文件:删除暂存区文件。
  3. git commit :删除版本库文件。

【为github创建SSH Key】
ssh-keygen -t rsa -C "youremail@example.com"：把邮件地址换成你自己的邮件地址，然后一路回车，使用默认值即可，由于这个Key也不是用于军事目的，所以也无需设置密码。创建SSH key，在~/.ssh目录生成id_rsa（私钥）和id_rsa.pub（公钥），并把公钥内容在github上设置好，你就可以push你的代码了。
cd ~进入用户主目录
ls -a 可以看到.ssh目录

【克隆远程仓库】
git clone git@github.com:账户/hello.git —> ssh协议,更快,推荐
git clone https://github.com/账户/hello.git —> https协议

【分支】
HEAD:指向的就是当前分支，切换到哪个分支，HEAD就指向哪个分支。
master:主分支
git checkout -b 分支名 :从当前分支或主分支创建分支并切换到分支
等效于
git branch 分支名 ：从当前分支创建分支
git checkout 分支名 ：切换分支

git branch：查看分支
git merge 分支名：合并指定分支到当前分支
git merge --no-ff -m "commit描述" 分支名：将分支名merge到当前分支，并且新创建commit
git branch -d 分支名：删除分支
git branch -D 分支名：强制删除分支。如果分支有commit但是没有merge（如果有merge到任意分支，普通删除就行），普通删除分支是不允许的。

git log --graph --pretty=oneline --abbrev-commit：查看历史commit记录，带图形显示

【保存现场】
切换到别的分支前，需要把在当前分支修改的commit或者stash，不然你在当前分支修改的切换到别的分支也能看到你修改的。
git stash：把当前未commit(但可以是git add过并且在恢复现场后需要重新git add)的工作现场给隐藏起来，可以多次stash，保存完现场后然后执行git status会看到之前的修改全部clean了。
git stash list：查看当前分支隐藏的工作现场，可能会有多个，以stash@{n}显示。
git stash apply stash@{n}：恢复指定隐藏现场
git stash drop stash@{n}：删除指定隐藏现场
git stash pop：恢复最近一次隐藏现场并且删除它

【merge某次commit】
git cherry-pick 4c805e2：将4c805e2次commit的内容（不是整个分支内容）复制到当前分支，可能有冲突。（在执行之前如果当前分支有改动需要将当前分支commit或stash）

【远程库】
origin：远程仓库的默认名称。
git remote -v:查看远程库的信息
git push origin master：把已经存在并提交的主分支推送到远程origin仓库。
git push origin 本地其他分支名：把本地已经存在并提交的分支推送到远程仓库分支。若远程服务器没有该分支，第一次push到远程服务器，远程服务器新创建一样的分支;若本地分支名与远程分支已经链接，则将推送更新相应的分支。
git pull：从远程库拉取当前所链接对应的分支。若第一次拉取，需要将远程拉取的分支名与当前所在的分支名进行链接：git branch --set-upstream-to=origin/远程分支名 本地分支名 （远程分支名与本地分支名需要相同，不然push会创建新的分支）
再进行拉取，可能有冲突，解决冲突，git add和commit，然后git push origin 相应分支名，推送到远程服务器。

坑：在关闭终端后隔了一天（有时候可能出现），远程分支有被修改，重新打开终端，修改本地分支，add、commit、push虽然会提示pull，但是pull后代码并没有更新，需要重新”git branch –set-upstream-to=origin/远程分支名 本地分支名“来建立链接，重新pull才回更新代码。

【创建标签】
git tag 标签名:为最新一次commit创建tag
git tag 标签名 commit号前几位：为特定commit创建tag
git tag -a 标签名 -m "tag信息" commit号前几位：为特定commit创建tag并添加附加信息

git show 标签名：查看标签信息

【删除标签】
git tag -d 标签名：删除本地标签
删除远程标签：
1.git tag -d 标签名：删除本地tag
2.git push origin :refs/tags/标签名：推送到远程删除标签

【标签推送远程】
git push origin 标签名：将某标签推送远程
git push origin --tags：一次性推送全部尚未推送到远程的本地标签

  1. 可以在远程对应的tag看到相应的推送代码，但是在相应的分支却没有更新代码，也就是说可以不git push到远程也能把tag push到远程。
  2. 假如tag有v1，v2，v3，本地删除v3（并没有删除v3所对应的修改），然后再推送全部tag到远程，在远程tag中只能看到v1，v2对应的代码；但是这时本地再创建tag v4并推送到远程，在远程可以在tag的v4看到包含v3的代码。

【忽略特殊文件】
在工作区的根目录创建.gitignore文件,将要忽略要上传的文件比如exe文件。注意：.gitignore文件需要commit并push到远程库。
.gitignore文件规则简单使用如下：

对于各种忽略文件可以在线查看：https://github.com/github/gitignore

添加某个被忽略的文件：
git add App.class
或者 git add -f App.class：强制添加
git check-ignore -v App.class：检查App.class文件是否被忽略，并显示在.gitignore文件忽略App.class文件的行数

【配置文件】
配置Git的时候，加上–global是针对当前用户起作用的，如果不加，那只针对当前的仓库起作用。
每个仓库的Git配置文件放在.git/config，也可以git config --local --list 查看
当前用户的Git配置文件放在/.gitconfig，也可以git config --global --list 查看
比如设置name和mail可以在/.gitconfig中看到相应值：
git config --global user.name "Your Name"
git config --global user.email "email@example.com"
