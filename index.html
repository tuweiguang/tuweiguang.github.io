搭建个人博客网站

发表于 2019-09-23

前言

首先作者近一年多来都有在CSDN平台写博客，之前就一直想尝试搭建一个个人博客，然后再上面写写小东西。终于经过两天的折腾，把博客网站搭建起来了。主要是借助Hexo和Github Pages：

  • Github Pages是由 GitHub 官方提供的一种免费的静态站点托管服务，让我们可以在 GitHub 仓库里托管和发布自己的静态网站页面。
  • Hexo 是一个快速、简洁且高效的静态博客框架，它基于 Node.js 运行，可以将我们撰写的 Markdown(.md文件) 文档解析渲染成静态的 HTML 网页。
    简单地说就是：
    markdown文件 –> 经过Hexo转换为html文件，上传 –> Github

搭建环境

作者本地环境：CentOS7 64位。
上面也说到了，搭建Hexo需要依赖nodejs环境和Git工具。

安装Git

很简单：yum install git
Git工具若不会，请参考廖雪峰的Git教程。

安装nodejs

  1. nodejs官方下载地址：https://nodejs.org/en/download/
  2. 下载：wget https://nodejs.org/dist/v10.16.3/node-v10.16.3-linux-x64.tar.xz
  3. 参照nodejs官方安装流程安装即可。

安装Hexo

  1. 安装
    npm install hexo-cli -g：这个过程可能很慢，作者在家安装好几次才安装成功，但是公司一下就好了
  2. 快速开始
    hexo init blog：初始化blog
    cd blog
    hexo install：安装组件
    hexo server：启动本地预览，登录本地网页查看
    若以上都没问题，说明Hexo已经安装完成，接下来就可以定制化自己的博客网站了。

  • 在blog目录中，source是源文件存放的位置，比如我们markdown文件。可以通过命令hexo new "name"插入新的md文件，并且在source相应目录下生成，当然你也可以直接把md文件拷贝进去。

细心地你肯定已经发现，在blog/themes/next目录下也有一个和站点配置文件一样的名字_config.yml，这也是一个配置文件，但是这个是主题的配置文件，把它叫做主题配置文件。这个主题配置详情更参考：https://www.jianshu.com/p/9f0e90cc32c2 。

发布到Github

发布博客：

  1. 先清空上次缓存：hexo clean。
  2. 将md文件生成html页面：hexo g。
  3. 本地预览：hexo s
  4. 发布到Github：hexo d
    （第2步和第4步合为一步：hexo g -d）
    发布成功可以去github仓库看看是否推送成功，最后访问 https://你的Github名字.github.io 进入你的博客。
